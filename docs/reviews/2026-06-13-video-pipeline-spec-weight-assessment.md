# VIDEO PIPELINE 規格重量與成熟度評估

日期：2026-06-13  
用途：提供其他 Agent 或工程／創作協作者進行獨立檢視與討論  
立場：保留細緻規格，但重新建立優先級、供給限制與硬性阻擋

## 討論目標

VIDEO PIPELINE 的長期目標不是單純自動串接素材，而是：

> 根據實際素材供給，產生語意正確、故事成立、觀看順暢，且可被人類剪輯師繼續修改的長片初剪。

短期合理目標應是「穩定產生可交付初剪」，而不是立刻取代熟練剪輯師。

本文件希望回答：

1. 現有規格是否過重？
2. 規格細度是否真的改善剪輯結果？
3. 目前失敗主要來自素材、規格，還是執行能力？
4. Pipeline 距離可用初剪與人類剪輯品質各有多遠？
5. 下一階段應保留、刪除或重新排序哪些規格？

## 評估背景

本次以同一批訓練素材比較：

- 學員人工剪輯版：約 9:24
- VIDEO PIPELINE Agent 版：約 10:40

相關詳細視覺 REVIEW：

- `docs/reviews/2026-06-13-student-vs-agent-montage-review.md`

Agent 版已能完成故事章節、旁白、MV 課程段落、素材地圖、剪輯、字幕與 VERIFY 產物，但觀看體感仍明顯落後學員版。

## 核心判斷

差距確實證明規格細度重要，但目前主要瓶頸已不是「規格不夠細」，而是：

1. 規格之間沒有足夠清楚的優先級。
2. 高優先級規則沒有形成真正的硬性阻擋。
3. 素材地圖只回答是否找到素材，沒有估算供給量與可支撐片長。
4. 系統會為了滿足片長與章節完整度，破壞素材語意與剪輯品質。
5. VERIFY 能發現部分問題，但發現後仍可能交付。

因此，現有規格不是單純「太重」，而是部分規格重量沒有轉化成決策權。

## 支持保留細緻規格的論點

現有細緻規格已帶來可驗證的進步：

- Agent 能區分故事口白段與課程 MV 段。
- 能建立完整的長片章節與情緒方向。
- 能產生素材地圖並識別 GAP。
- 能輸出節奏、字幕、素材重複與視覺稽核證據。
- 能將剪輯失敗拆解成素材錯配、重複、節奏與故事問題。
- 能使同一次生產過程具備可追蹤、可比較、可修改的 artifact。

若移除這些規格，Pipeline 很可能退回「能產生影片，但難以解釋與修正」的狀態。

細規格的主要價值不是保證成片一定好，而是：

> 讓創作意圖與失敗原因能夠被系統化表達、驗證與迭代。

## 認為規格過重的合理論點

「規格太重」並非完全錯誤。現況確實存在以下風險：

- 規格項目很多，但模型可能無法同時可靠遵守。
- 大量欄位增加生成、轉換與維護成本。
- 規格形式完整，可能讓系統誤以為品質已成立。
- 下游可能只選擇容易量化的要求執行，例如片長、段落數與字幕。
- 多層轉換容易使原始創作意圖逐步失真。
- 太多軟性規則若沒有優先級，最後等同沒有規則。

本次 Agent 版就是具體案例。系統成功滿足：

- 超過 10 分鐘
- 有完整章節
- 有旁白
- 每段都有畫面

但同時違反：

- 畫面語意必須正確
- proof 類內容不得錯配
- 素材不足時必須縮短
- 每刀應提供新的視覺資訊
- VERIFY 失敗不得交付

這代表規格不是越多越好。沒有決策優先級的規格，會產生形式合規但品質失敗的結果。

## 本次失敗的客觀拆解

### 素材限制

- 部分課程缺少足夠原始素材。
- 活線作業沒有合格可用來源。
- 許多段落只有單一視角或照片。
- 結尾缺少出發、交接、告別與人物反應素材。

素材限制確實降低了 Agent 可達成的上限。

### Pipeline 規劃問題

- 在素材供給評估前，就承諾 11–13 分鐘的劇本。
- 將「至少 10 分鐘」視為硬要求。
- 沒有先計算每個章節需要多少影片、照片與有效鏡頭。
- 沒有回推每段最大誠實片長。

### Pipeline 執行問題

- 原始素材地圖已正確把活線作業標成 GAP，但下游繞過 GAP。
- 沒有提示詞、沒有 caption、匹配分數為零的素材仍被接受。
- 同一來源被切割成多個窗口，並被當成多樣素材。
- 使用固定秒數換鏡，缺乏動作頓點與內容頓點。
- VERIFY 已判定素材重複失敗，成片仍然完成交付。

### 人類剪輯能力差距

學員版較能辨識：

- 素材內已完成的特效與內容橋段
- 動作開始、受力、完成與反應
- 課程之間的自然轉場
- 哪些畫面應短切、哪些畫面應停留
- 如何利用有限素材建立人、事、時間與情緒推進

這些仍是目前 Agent 的弱項。

## 規格應重新分層

建議將所有規格分成三層，並禁止低層要求破壞高層要求。

### 第一層：不可違反

- 畫面語意正確
- 素材來源誠實
- proof／identity／testimony 類內容不得錯配
- 零分匹配不得使用
- GAP 不得被無聲繞過
- 素材不足不得以重複內容硬補片長
- VERIFY 阻擋錯誤不得交付

### 第二層：品質目標

- 故事具備問題、壓力、轉折與 payoff
- 動作段落包含建立、動作、細節、結果與反應
- 鏡頭長度有節奏變化
- 每次換鏡應提供新資訊或明確情緒功能
- 關鍵段落應對齊素材內的動作與特效頓點

### 第三層：風格偏好

- 目標片長
- 特效形式
- 字幕、字卡與視覺風格
- 轉場偏好
- 音樂類型
- 旁白聲線

執行原則：

```text
風格偏好不得破壞品質目標。
品質目標不得破壞素材誠實與語意正確。
```

## 素材地圖應承擔的責任

素材地圖應在正式劇本前產生，而且不只回答「有沒有素材」。

每個故事或課程段落應包含：

```json
{
  "requested_duration_sec": 30,
  "target_shot_sec": 2.0,
  "required_effective_shots": 15,
  "required_unique_sources": 8,
  "required_functions": [
    "establish",
    "preparation",
    "action",
    "detail",
    "result",
    "reaction"
  ],
  "available_video_sources": 2,
  "available_photos": 3,
  "estimated_effective_shots": 7,
  "max_honest_duration_sec": 16,
  "feasibility": "thin",
  "recommended_action": "shorten_or_merge"
}
```

素材地圖因此也必須能指出：

- 編劇規劃是否超出素材供給
- 導演是否規劃了未拍攝的內容
- 攝影是否缺少建立、動作、細節、結果或反應鏡頭
- 真實可支撐片長是多少

## 對「規格太重」的建議處理

不建議直接刪除大量規格。建議採取：

### 保留

- 能影響素材誠實、故事結構與 VERIFY 的欄位
- 能支持錯誤追蹤與局部重建的 artifact
- 素材需求量、鏡頭功能與最大誠實片長

### 合併

- 重複表達相同意圖的欄位
- 只在不同節點重新命名、但不增加決策價值的欄位
- 無法被 BUILD 或 VERIFY 消費的描述性欄位

### 降級

- 無法穩定執行的風格偏好
- 不應阻擋交付的主觀美學要求
- 在素材不足時必須讓步的目標片長

### 移除

- 沒有消費者、沒有驗證方式、也沒有人工用途的欄位
- 只為了讓 SPEC 顯得完整而存在的欄位

判斷某條規格是否值得保留，可詢問：

1. 哪個節點會使用它做決策？
2. 哪個 VERIFY 能判斷它是否成立？
3. 違反時，流程會採取什麼動作？

若三題都無法回答，該規格很可能只是文件重量。

## 成熟度評估

以下為基於本次案例的粗略判斷，不是精密評分：

| 能力 | 估計成熟度 |
|---|---:|
| 工程流程完整度 | 70–80% |
| 長片組裝與自動化 | 65–75% |
| 可追蹤 artifact 與稽核 | 60–75% |
| 素材語意可靠度 | 40–50% |
| 素材供給與劇本協調 | 25–40% |
| 人類式局部剪輯判斷 | 25–35% |
| 整體導演與情緒控制 | 25–35% |

## 距離目標還有多遠

### 距離「可交付初剪」

不算很遠。

若優先完成以下事項，品質應能明顯提升：

1. 劇本前先計算素材供給與可支撐片長。
2. 零分素材、語意錯配與 proof 錯配直接阻擋。
3. 素材不足時縮短或重寫，不得硬補片長。
4. 將高密度逐章與關鍵段落蒙太奇納入 VERIFY。
5. VERIFY 失敗時停止交付。

### 距離「接近熟練剪輯師」

仍有相當距離。

需要進一步具備：

- 素材內動作頓點辨識
- 既有特效與內容橋段理解
- 音樂、動作與情緒同步
- 對人物反應與故事 payoff 的判斷
- 從大量人類成片中學習局部剪輯模式
- 能主動放棄原始劇本並重構故事

這部分不是增加更多文字規格就能直接解決，需要更好的素材理解、比較證據與迭代資料。

## 建議的近期目標

下一階段不應以「增加更多創作規格」為主，而應以「讓現有重要規格真正掌握決策權」為主。

建議近期成功標準：

```text
即使影片變短，也不使用錯誤素材。
即使章節減少，也不重複素材硬補時間。
即使故事較簡單，也確保每個段落都有正確且足夠的視覺證據。
```

若能穩定達成以上三點，Pipeline 將從「能完整跑完」進入「能產生可信初剪」。

## 希望其他 Agent 挑戰的問題

請針對以下問題提出支持、反對或替代方案：

1. 三層規格優先級是否足夠，還是應採用其他模型？
2. 哪些現有規格應被刪除，而不只是降級？
3. 素材地圖應如何估算有效鏡頭數與最大誠實片長？
4. 如何避免把同一來源的多個窗口錯算成多樣素材？
5. 動作頓點與既有特效橋段應在 ingest、curation 還是 edit 階段辨識？
6. 哪些 VERIFY 應屬於硬性阻擋，哪些只能提供建議？
7. 可交付初剪的最低品質門檻應如何定義？
8. 是否有比重規格更有效的方法，能提升 Agent 的人類式剪輯判斷？

## 我的最終立場

你的規格方向整體正確，不應因為「太重」而直接大幅刪除。

但目前確實需要停止持續增加同等權重的規格，改為：

1. 建立清楚優先級。
2. 將高優先級規格轉成硬性阻擋。
3. 刪除沒有決策者與 VERIFY 的規格。
4. 讓素材供給決定劇本承諾與最終片長。
5. 用人類成片對照與高密度視覺證據改善局部剪輯能力。

簡短結論：

> 距離可信、可交付的自動初剪不算很遠；距離穩定接近熟練剪輯師仍有一段長路。規格細度是必要基礎，但下一次主要進步不會來自更多規格，而會來自更正確的優先級、阻擋機制與素材理解。
