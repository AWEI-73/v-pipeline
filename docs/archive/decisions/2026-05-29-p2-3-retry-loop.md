# Decision: P2-3 Self-reflection Retry Loop + P1-3 Decision Log

Date: 2026-05-29
Status: verified
Scope: video_pipeline.py / pipeline() orchestrator
Superpowers phase: execute

## SPEC

Requirement:
pipeline() QA 失敗時只回傳 `qa_fail` + exit 1，沒有任何重試。roadmap 的靈魂是「VERIFY 用 fix_target 指路重跑」。需要實作 self-reflection retry loop，並記錄每輪決策軌跡（P1-3）。

Why:
- seg10「將散未散」等抽象語意段落 prepick 全 no，但整體影片可能因其他段落拉高平均而 pass，不觸發 retry —— 這是設計使然（retry 只在整片 fail 時觸發）
- P1-3 的決策軌跡（picks/gate/QA/fix_actions）目前散在多個 JSON 檔，無統一軌跡

Direction:
- Retry 預設開啟，`--no-retry` 關閉；`max_retries=2`
- 重選策略：換下一個 survivor + 用 8b 複核（`vlm_model_retry=qwen3-vl:8b`）
- 候選耗盡 → 標 `unfixable`（誠實放棄，不無限重試）
- Decision Log：每輪快照寫入 `decision_log.json`

Non-goals:
- 不做逐段 gate（D1，deferred）
- TTS/SRT/mix-audio 在 loop 外只跑一次（與素材無關）
- 不自動 push 到任何遠端

## DO

Files / modules:
- `video_pipeline.py` 唯一修改檔

Function-level plan:
- `render_one(n, script, candidates, picks, actual_dur, xfade, mat, verbose, fresh=False)`: 抽出單段渲染，`fresh=True` 清舊快取（重試必須下載新素材）
- `repick_segment(n, seg, exclude, thumbs_dir, model, verbose)`: 對非 exclude 候選跑 prepick_vlm_filter（8b），全否決回 None（ceiling）

- `compose_and_qa(...)`: 可重複呼叫的 steps 5b-11（precompose_gate → xfade → merge → thumbnails → verify → content_qa → 重讀 qa_report）
- `pipeline()` 主迴圈：attempt=0 跑首輪，while not qa.pass and attempt < max_retries: collect_fix_actions → repick → render_one(fresh) → compose_and_qa → snapshot
- `_snapshot()` 內部函式：每輪把 picks/gate/qa/content_alignment/fix_actions 存入 `attempts[]`
- 結尾寫 `decision_log.json`

Data / interface changes:

- pipeline() 回傳新增：`attempts_used`, `unfixable_segments`, `decision_log`
- 新 CLI 參數：`--no-retry`, `--max-retries N`, `--vlm-model-retry`

Migration / compatibility:

## Decision Notes

Accepted because:
- 換 survivor + 8b 複核：成本與成功率平衡；比「直接升 8b 重搜」便宜，比「只換下一個」可靠
- max_retries=2：VLM 每輪很慢（~3-5min），2 輪已足夠；seg10 這類 ceiling 案例候選耗盡後就 unfixable 不再重試
- 預設開啟：符合「整條 pipeline 靠 VERIFY 自省」的設計初衷

Tradeoffs:
- 每輪 retry 需完整重跑 xfade concat + merge-final（慢）。只重渲變動段，但 xfade 鏈必須全連。
- 單段爛分被平均稀釋（D1）：content_alignment 是平均分，整體 pass 就不觸發 retry，個別爛段被放過 —— deferred

Open questions:
- D1：若要修，加 `--strict` 模式：任一段 score<60 強制 repick（不看整體）
