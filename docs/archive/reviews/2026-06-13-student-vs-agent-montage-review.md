# 學員版與 Agent 版蒙太奇剪輯 REVIEW

日期：2026-06-13  
比較影片：

- 學員版：`最終版的最終版.mp4`，約 9:24
- Agent 版：`20260612-232948-story-mv/final.mp4`，約 10:40

## REVIEW 目的

本報告不是判斷哪一版「比較漂亮」，而是找出兩版觀看體感差異的具體來源，並確認 VIDEO PIPELINE 下一步應優先改善的能力。

本次採用四層證據：

1. 全片 48 格鳥瞰蒙太奇
2. 六個章節的並排蒙太奇
3. 逐鏡節奏條與場景切換統計
4. 開場、課程、生活、結尾的高密度關鍵段落蒙太奇

## 結論摘要

學員版明顯比較像人剪，主要原因不只是素材比較多，而是剪輯者有辨識：

- 畫面本身已設計好的特效與內容橋段
- 動作開始、撞擊、完成與反應等剪輯頓點
- 哪些畫面只能短暫出現，哪些畫面值得停留
- 課程、生活、回顧與告別之間應如何換氣
- 同一段落中需要建立、動作、細節、結果與反應等不同功能的鏡頭

Agent 版已能建立完整旁白與章節結構，較早期結果有明顯進步；但目前仍把「段落時間分配」誤當成「剪輯節奏」。當素材不足時，它傾向延長同一來源、固定秒數換鏡、重複素材，而不是縮短段落或重寫劇本。

因此主要差距不能完全歸因於素材。素材不足放大了問題，但更核心的問題是 Agent 尚未充分理解素材中的剪輯意圖與有效頓點。

## 第一層：全片鳥瞰

### 學員版

![學員版全片鳥瞰](assets/2026-06-13-montage-verify/overview_student_48.jpg)

全片能看見明確推進：

`標題與班級建立 → 訓練日常 → 多項課程 → 生活與情感 → 主任／回顧 → 去向與感謝`

即使部分段落設計較傳統，畫面內容持續改變，觀眾能感受到課程種類、人物關係與時間推進。

### Agent 版

![Agent 版全片鳥瞰](assets/2026-06-13-montage-verify/overview_agent_48.jpg)

Agent 版的主要問題在鳥瞰層就能看見：

- 開頭長時間停留於空拍與坐姿合照
- 中段同一訓練畫面反覆延長
- 生活段落主要由少量合照輪流支撐
- 後段回到前面已使用過的素材
- 結尾再次使用長時間空拍，缺少新的情緒證據

旁白在推進故事，但畫面沒有同步提供新的資訊，因此觀看感受會拖、散、不順。

## 第二層：逐章比較

### 開場

![開場逐章比較](assets/2026-06-13-montage-verify/chapter_01_opening_compare.jpg)

學員版在約 80 秒內完成標題特效、班別群像、晨間運動、安全承諾、集合與準備。每一組畫面都在回答新的問題：

- 這是誰的故事？
- 他們從哪裡開始？
- 日常訓練是什麼樣子？
- 團隊與紀律如何建立？

Agent 版前段主要使用空拍、坐姿合照與安全宣示。旁白意圖正確，但視覺論述只靠少數來源，沒有形成完整開場橋段。

### 課程動作

![課程逐章比較](assets/2026-06-13-montage-verify/chapter_03_course_action_compare.jpg)

學員版依序呈現器材、操作、合作、高空作業、細節與成果，形成「看懂一項工作」的最小敘事。

Agent 版有部分正確課程畫面，但同一活動停留時間過長，且課程之間缺少清楚的完成點與轉場邏輯。活線作業段落另有嚴重素材錯配，應視為 VERIFY 阻擋問題。

### 選擇與轉折

![選擇逐章比較](assets/2026-06-13-montage-verify/chapter_05_choice_compare.jpg)

學員版利用工作現場、人物互動、塔架與去向卡片建立「訓練即將轉化成職涯」的轉折。

Agent 版以課堂與集合畫面敘述相同概念，但視覺證據較弱，且前段合照再次出現，削弱了轉折的新鮮感。

### 告別與結尾

![結尾逐章比較](assets/2026-06-13-montage-verify/chapter_06_farewell_compare.jpg)

學員版的結尾雖偏簡報式，但有清楚的人物去向、教師感謝與公司識別，完成資訊收束。

Agent 版把去向、團體照與空拍重新排列，但缺少新的結果畫面與人物反應，導致結尾像再次總結前文，而不是情緒落點。

## 第三層：關鍵段落高密度蒙太奇

高密度蒙太奇比逐格分析節省成本，仍能看出橋段、頓點、來源重複與視覺推進。建議未來將此層作為重要段落的標準 VERIFY 證據。

### 開場設計、特效與內容橋段

![開場高密度比較](assets/2026-06-13-montage-verify/key_opening_detail_compare.jpg)

學員版開場最值得學習。標題不是獨立裝飾，而是搭配素材內容完成「最後一頁／即將出發」的命題；接著以群像、班別、晨間運動、安全宣示逐步落地。

Agent 版的空拍本身品質不差，但連續使用約 40 秒，之後又長時間使用坐姿合照與安全宣示。畫面之間缺少問題、回答、升級與轉折，因此口白再完整也難以產生張力。

### 課程動作與剪輯頓點

![課程高密度比較](assets/2026-06-13-montage-verify/key_course_detail_compare.jpg)

學員版能看出大量有效頓點：

- 器材或場域建立
- 動作起始
- 受力、敲擊、拉動或攀爬
- 工具與手部細節
- 完成狀態
- 下一課程的視覺橋接

Agent 版則經常把同一個活動、站位或結果畫面持續切割。這種剪法雖然有 cuts，卻沒有新的動作資訊，因此觀眾感受不到節奏推進。

### 生活與情緒轉折

![生活段落高密度比較](assets/2026-06-13-montage-verify/key_life_detail_compare.jpg)

學員版使用比賽、生日、遊戲、合照、主任談話、空拍與工作畫面，形成從歡樂到回顧的轉折。

Agent 版主要由三組合照與早期坐姿合照支撐。素材不足確實是限制，但系統應選擇縮短段落，而不是持續重複靜態畫面。

### 結尾與 payoff

![結尾高密度比較](assets/2026-06-13-montage-verify/key_ending_detail_compare.jpg)

學員版結尾利用人物去向卡片形成資訊 payoff，最後以教師感謝與公司識別收束。

Agent 版有嘗試引用去向卡片與團體照，但後續又回到長時間空拍。結尾的旁白命題是「帶著責任出發」，畫面卻沒有足夠的出發、交接、人物反應或新工作場景作為證據。

## 根因分類

### 素材限制

- 部分規劃課程沒有足夠原始素材
- 很多素材只有照片或單一視角
- 活線作業沒有合格可用來源
- 結尾缺乏出發、交接、告別與人物反應素材

### 素材地圖不足

現行素材地圖主要判斷「是否找到素材」，沒有在寫劇本前計算：

- 預計段落秒數需要多少鏡頭
- 影片、照片與唯一來源各需要多少
- 是否具備建立、動作、細節、結果、反應等鏡頭功能
- 素材最多能誠實支撐多少秒
- 缺件是否代表導演規劃或拍攝規劃不足

### Agent 剪輯判斷不足

- 沒有充分辨識素材內已完成的特效橋段
- 沒有依動作頓點選擇剪輯點
- 把固定鏡頭秒數當作節奏
- 把同一來源的多次切割視為素材多樣性
- 缺少對「新資訊是否出現」的 VERIFY
- 素材不足時不會主動縮短段落與重寫劇本

## 建議的 VERIFY 升級

未來不應只產生一張全片 12 格縮圖。建議預設輸出：

| 層級 | 建議密度 | 用途 |
|---|---:|---|
| 全片鳥瞰 | 36–48 格 | 故事推進、重複與章節比例 |
| 逐章蒙太奇 | 每章 12–16 格 | 比較各章內容供給與視覺邏輯 |
| 關鍵段落蒙太奇 | 每段 24–40 格 | 橋段、動作頓點、特效、錯配 |
| 節奏條 | 全片所有偵測鏡頭 | 鏡頭長度、節奏變化與長停留 |

此外，每個課程或故事段落應新增以下 VERIFY：

- `new_visual_information_ratio`
- `unique_source_ratio`
- `action_phase_coverage`
- `repeated_visual_hold_sec`
- `max_honest_duration_sec`
- `script_overreach`
- `wrong-proof-material`

## 對素材地圖的直接要求

素材地圖應在劇本前完成供需估算。例如 30 秒課程 MV、目標每鏡 2 秒，不能只判斷「有影片」；它需要至少約 15 個有效鏡頭，並包含不同鏡頭功能。

若只有 2 支影片與 3 張照片，系統應回報：

```text
規劃需求：15 個有效鏡頭
可用供給：約 7 個有效鏡頭
最大誠實片長：約 14–18 秒
建議：縮短段落、合併相近課程，或要求補拍
```

素材不足不應自動轉化為重複素材與延長片長。

## 最終判斷

學員版的優勢不是單純「人工有美感」，而是它能辨識素材自身的設計與動作語法，並根據內容調整節奏。Agent 版目前已具備組織長片、建立旁白與輸出完整流程的能力，但尚未能穩定完成具有人類剪輯感的局部決策。

下一階段應優先改善：

1. 劇本前的素材數量與鏡頭功能估算
2. 素材內動作頓點與既有特效橋段辨識
3. 依素材供給縮短或重寫段落
4. 高密度逐章與關鍵段落蒙太奇 VERIFY
5. 對視覺重複、錯誤證據與無新資訊鏡頭進行阻擋

本次 Agent 成片應判定為：結構與輸出流程有進步，但剪輯品質 VERIFY 不通過。
