# 影片腳本腦暴 → 素材缺口 → 拍攝指令工作流

## 1. 這份模板的目的
這份模板的目的是：

1. 先由懂影片風格的 Agent 做腳本腦暴
2. 再根據腳本反推出素材缺口
3. 把缺口寫成夠具體的拍攝指令
4. 交給學員補拍 / 補攝 / 補圖
5. 由 Agent 與 Subagent 完成字幕、音樂、整合剪輯
6. 最後由你做最終 Review

這不是「先亂拍再看怎麼剪」的流程，而是：

> **先定腳本方向，再定素材方向，再定拍攝任務。**

---

## 2. 角色分工

### 2.1 你：Reviewer / Spec Owner
你負責：
- 定義影片主題
- 定義影片要傳達的感受
- 定義成功標準
- 最後 Review 成片

你不是執行剪輯的人，你是最後判斷是否通過的人。

### 2.2 Agent：主執行者
Agent 負責：
- 依照目前架構做腦暴
- 選腳本方向
- 判斷哪裡缺素材
- 規劃字幕 / 音樂 / 畫面節奏
- 整合素材並完成剪輯

Agent 是真正把影片做出來的人。

### 2.3 Subagent：專項處理者
Subagent 可以被派去做：
- 素材整理
- 字幕草稿
- 音樂搜尋或生成
- 文案 / 旁白草稿
- 缺口盤點
- 驗證清單

### 2.4 學員：素材收集者
學員只負責：
- 按照指令拍照 / 攝影
- 補拍指定情境
- 交出原始素材

學員不負責核心剪輯決策。

---

## 3. 正確流程順序

```text
主題與架構確認
  ↓
Agent 腳本腦暴
  ↓
選定 1 個腳本方向
  ↓
反推素材缺口
  ↓
寫成具體拍攝指令
  ↓
學員補拍 / 補圖
  ↓
Agent + Subagent 做字幕 / 音樂 / 整合
  ↓
你做最終 Review
```

這個順序不能倒過來。

---

## 4. 腳本腦暴階段

### 4.1 目的
先看目前的架構與素材條件，想出這支片可以怎麼發揮。

### 4.2 需要輸入的資訊
Agent 在腦暴前至少要知道：
- 影片主題
- 目標受眾
- 目前已有素材類型
- 影片長度範圍
- 你想要的情緒或風格

### 4.3 腦暴輸出
Agent 要輸出：
- 2～3 個可行腳本方向
- 每個方向的優點 / 風險
- 推薦哪一個方向
- 為什麼推薦

### 4.4 腳本方向要寫得夠具體
不能只寫：
- 熱血一點
- 有訓練感
- 很有故事感

要寫成：
- 開場如何切入
- 中段如何承接訓練內容
- 哪一段需要補鏡頭
- 結尾如何收束

---

## 5. 由腳本反推素材缺口

### 5.1 目的
確認腳本成立所需要的畫面，哪些已經有，哪些還沒有。

### 5.2 缺口分類
常見缺口分成幾類：
- 動作鏡頭缺口
- 情境鏡頭缺口
- 空景 / 過場缺口
- 特定表情缺口
- 字幕對應缺口
- 音樂情緒缺口

### 5.3 缺口表要怎麼寫
每一筆缺口都要包含：
- 缺什麼
- 用在哪一段
- 需要幾秒
- 要什麼鏡位
- 要什麼情緒
- 要不要補拍
- 由誰負責補

### 5.4 範例
不是只寫：
- 要有跑步畫面

而是要寫：
- 需要 1 段學員跑步畫面，作為訓練進行中的過渡鏡頭
- 鏡頭長度 5～8 秒
- 需要正面或側面中景
- 畫面要看得出是訓練中的移動感
- 用來接「上課」段落與「實作」段落之間

---

## 6. 拍攝指令階段

### 6.1 目的
把素材缺口轉成學員可執行的拍攝任務。

### 6.2 拍攝指令必填欄位
每一條指令至少要寫：
- 拍什麼
- 幾個人
- 什麼動作
- 什麼場景
- 什麼鏡位
- 什麼時長
- 要補哪一段
- 為什麼要拍

### 6.3 好的指令範例
- 拍 3～5 位學員在走廊中慢跑 1 段
- 鏡頭要有中景，至少 5 秒
- 需要看出訓練氛圍，不要像日常散步
- 用來補「課堂切換到實作」的轉場

### 6.4 不好的指令範例
- 拍一下跑步
- 幫我補點上課畫面
- 再拍一些有感覺的素材

這些都太模糊，不能當執行指令。

---

## 7. 字幕與音樂也可以由 Agent / Subagent 處理

### 7.1 字幕
Agent 可以交給 Subagent 做：
- 字幕草稿
- 重點句標記
- 台詞切分
- 需要強調的字詞

### 7.2 音樂
Agent 可以交給 Subagent 做：
- 音樂搜尋
- 音樂風格建議
- 音樂生成草稿
- 情緒對位檢查

### 7.3 原則
字幕與音樂不是最後隨便補上去的。
它們要跟腳本方向同步決定。

---

## 8. 最終 Review 標準

Review 不看「好不好看」而已，而是看：
- 腳本方向有沒有選對
- 素材缺口有沒有補準
- 拍攝指令有沒有夠具體
- 字幕與音樂有沒有對位
- 成片能不能直接講清楚主題

### 8.1 具體檢查問題
1. 這支片的方向是否明確？
2. 選定的腳本是否符合目前架構？
3. 素材缺口是否真的被補到？
4. 拍攝指令是否足夠詳細？
5. 字幕與音樂是否一致？
6. 成片是否還有模糊空間？

### 8.2 通過條件
- 內容具體
- 角色分工正確
- 素材對位清楚
- 沒有需要猜的地方
- 可以直接往下一步走

---

## 9. 一句話版本
> **先由 Agent 腳本腦暴，再由腳本反推素材缺口，最後把缺口寫成具體拍攝指令，交給學員補拍，Agent 再統整字幕、音樂與成片。**
