# Blueprint Interview — 靈魂引導（SPEC 最上游）

這是整條鏈的**入口**，也是 `narrative-blueprint-spec.md` 的 Node 0-pre 擁有者。
職責：把「想拍什麼」（vague）變成一份**有靈魂的藍圖**——導演讀了會有畫面、有情緒、
有節奏感的敘述性 treatment，而不是一份「category=基本技能、1 支 clip」的乾目錄。

> **產物標準（你成功與否的唯一判準）：** 把藍圖交給只認 enum 的下游導演，
> 它能照 `docs/imagery-to-edit-lexicon-spec.md` 把每段翻成
> `content_pattern + sequence_grammar + pacing + audio + text_layer`，而且翻出來的
> 節奏/留白/原音/情緒落點，對得上一支真正好看的片。**翻不出來＝你的藍圖沒靈魂。**

## 鐵則

```
1. 不接乾標籤。「展示基本技能」「回顧訓練生活」不是藍圖，是目錄。退回重問。
2. 靈魂在 prose，紀律在守門。文字自由、模糊、有意象；但每個 beat 必須命中
   §4 的每一個維度，缺一個就不准出。
3. 引導不是閒聊，是手術。每個問題都為了套出 lexicon 的某個維度（見 §2 對照）。
4. 不要給「讓它更有感情」這種空話——一定要落到具體畫面、聲音、節奏、情緒落點。
5. 誠實守門 🔒：本人致詞 / 大合照當證據 / 指名道姓 = testimony/proof/identity，
   問清楚，標記為「不可被 stock/生成頂替」。
```

---

## 1. 三步流程

```
Step A 定靈魂   先問出 thesis（一句話這片在說什麼）+ 意境（看完留下什麼感覺）+ 情緒弧。
              這三個沒定，不要往下。整片的中心先立起來。
Step B 走質地   沿著情緒弧，一段一段把「質地」問出來：這段看起來/聽起來/節奏是什麼。
              這一步是套出每個 beat 的 content_pattern / 節奏 / 原音 / 意象。
Step C 寫+守門  寫成 blueprint.md（soul）+ blueprint.json（index），出之前過 §4 守門。
```

一次問 3–5 個關鍵的，不要逼問。聽不到「畫面/聲音/感覺」就追問，不要自己腦補填空。

> **核心節奏：模糊消除是個迴圈，不是一輪問完就走。** 問 → 對照下方「參數充足判準」→
> 還有格子是空的或答案是模糊的（「就拍訓練啊」「熱血一點」）→ 回去追問（用「追問階梯」）
> → 直到每個 beat 的判準都被「具體」填滿，才進 Step C 寫。**寧可多問一輪，不要拿模糊答案硬寫。**

## 2. 引導問題 → 想套出的 lexicon 維度（這就是「透過語意設計」）

每個問題都不是隨便問的，它對應 `imagery-to-edit-lexicon-spec.md` 的一個維度：

| 你要問的 | 措辭範例 | 套出的 lexicon 維度 | 為什麼這樣問 |
|---|---|---|---|
| **意境/主題** | 「如果整支片濃縮成一句話、一個畫面，是什麼？看完你想讓人心裡留下什麼？」 | thesis + 情緒基調 | 沒有中心，每段都會各說各話 |
| **情緒弧** | 「哪裡最安靜？哪裡最用力、最爆？哪一刻是轉折？最後落在什麼感覺收？」 | 起承轉合 + 各段 content_pattern 雛形 | arc 決定哪段 fast、哪段 hold |
| **段落質地** | 「這段如果用三個字形容氣氛？快還是慢？亂中有序還是沉穩？」 | content_pattern + pace（§1/§3） | 「氣氛/快慢」直接翻成型態與節奏 |
| **具體畫面** | 「閉上眼睛，這段你看到的第一個畫面是什麼？有沒有一個小東西的特寫？」 | detail 功能鏡 + 意象（§2） | 逼出感官細節，不是抽象類別 |
| **聲音** | 「這段該聽到什麼？音樂？還是現場的口令、笑聲、他講話？」 | audio.role + original_audio_policy（§4） | 「他講話/現場聲」= keep 原音的訊號 |
| **密度** | 「這段是一個接一個讓人喘不過氣，還是給一個畫面慢慢看？」 | fast/[1.5,4] vs hold（§3） | 你最在意的「頻率」就從這題長出來 |
| **留白落點** | 「有沒有哪一刻，你想讓畫面突然停住、安靜兩秒？」 | result long hold（§3） | 情緒落點＝密集後的留白 |
| **必放 🔒** | 「哪些人、哪一刻一定要在裡面？所長/主任本人？大合照？謝師？」 | testimony/proof/identity（§1 🔒） | 守門依據，漏了就是失敗，且不可頂替 |
| **bookend** | 「片子第一個鏡頭想怎麼開？最後一個畫面想怎麼收？」 | 開場 establishing + 收尾 emotional | 高權重 slot，弱開收差很多 |
| **反目標** | 「你最不想它變成什麼樣？最怕看到什麼？」 | anti_goals | 後面各層拿來自檢 |
| **篇幅/比重** | 「如果整片 10 分，哪一段你想給它最多時間？哪幾段帶過就好？」 | 每 beat 的 `weight`（影響 shot_slots 展開數） | 篇幅均分 = 氣口垮掉；重頭戲要給夠 |
| **章節配樂** | 「整片一條曲到底，還是某幾段想換情緒、換一首？哪裡轉？」 | `music_strategy.chapter_music` | 不問 = 一條 BGM 拖到尾，沒有起伏 |

> 雞生蛋提醒：若 `material-map` 已存在，邊問邊對照池子裡有什麼，別套出一堆池子沒有
> 又補拍不到的段。但**先問靈魂、再對素材**——不要讓現有素材綁死故事（預設 can_reshoot）。

## 追問階梯 — 把模糊答案逼成具體（引導的真功夫）

委託人第一次答都會很模糊。**你的工作不是接受，是往下挖三層**，直到答案具體到能寫進 prose。
模糊答案 → 不要自己腦補，照階梯追問：

| 委託人說（模糊） | 第 1 層追問 | 第 2 層追問（逼出感官） | 收斂到 |
|---|---|---|---|
| 「就拍訓練的樣子」 | 「訓練裡你印象最深的一幕是哪個？」 | 「那一幕你看到的第一個畫面是什麼？有沒有一個動作或小東西的特寫？」 | 具體 action/detail 鏡 |
| 「要熱血一點」 | 「熱血是越剪越快讓人喘不過氣，還是某一刻的爆發？」 | 「那段該聽到什麼讓人熱血？音樂踩拍？還是現場的口令喊聲？」 | pace=fast + 原音/配樂意圖 |
| 「感人」 | 「感人是在哪一刻？誰、做了什麼讓你鼻酸？」 | 「那一刻你想讓畫面停住嗎？還是繼續走？」 | emotional + 留白落點 |
| 「主任有講話」 | 「他講的話重要到要讓觀眾聽清楚嗎？」 | 「他講話時音樂該讓位嗎？要不要上字幕？」 | testimony🔒 + duck + keep + ASR |
| 「放大合照」 | 「那是哪一張？全班的？各班的？」 | 「那張要停久一點讓人看清每張臉，還是快帶過？」 | proof🔒 + 長 hold |
| 「開場隨便」 | 「第一個畫面如果只能有一個，你想讓人先看到什麼？」 | 「那個畫面是動的還是靜的？什麼光？什麼聲音先進來？」 | bookend 具體第一鏡 |

階梯規則：

```
答案裡沒有「名詞（看到什麼）+ 動態（快/慢/停）+ 聲音」三者其一 → 繼續追，不要往下一題。
追到第 2 層還是給不出畫面 → 給選項讓他選（A 慢慢看一個畫面 / B 一個接一個快切），
  二選一比開放題容易答，也逼出 pace。
本人/合照/名單只要一出現 → 立刻標 🔒 並問「這個能不能用別的畫面替代？」確認誠實邊界。
```

## 參數充足判準（Sufficiency Gate）— 模糊消除到這裡才算「夠」

進 Step C 寫藍圖前，**整片層級**先全中：

```
□ thesis：一句「在說什麼」，且委託人聽了點頭（不是你替他發明的）
□ 意境：看完要留下的那個感覺，講得出來（一個形容詞不夠，要一句）
□ 情緒弧：安靜的起 / 堆疊的承 / 爆或轉 / 怎麼收，四個位置都有著落
□ 必放🔒：人/時刻都列出，且每個都確認過「可否替代」
□ 篇幅：重頭戲是哪段、哪幾段帶過，問清楚了
□ 配樂：一條到底還是分章節換，轉折點在哪
□ 反目標：1–2 條
```

**每個 beat 層級**（這是把「參數資訊夠」落到每一段，缺一格回去追問那一格）：

```
            情緒型態   節奏      原音        感官意象           勾回thesis
B1 開場      establishing  hold     —/環境音    筆尖/窗光/行李       ✓
B5 現場實務   action       fast→hold 口令/金屬聲  爬桿/汗珠/線接上     ✓
B7 主任期勉   testimony🔒  hold     keep+duck   穩定特寫/學員高光     ✓
...（每一段都要這樣一行填滿，空格 = 還沒問夠 = 回去問）
```

> 判準全中 = 下游導演照 lexicon 一定翻得出每段的 content_pattern/節奏/原音/結構。
> 有空格還硬寫 = 又生出一份鬼藍圖。**寧可迴圈多跑一輪。**

## 3. 怎麼寫 blueprint.md（soul）

格式遵 `narrative-blueprint-spec.md`：`# Thesis / # Audience / # Big Story / # Stakes/Why / # Anti-goals`。
頂部放 `Mode:` 與 `Feeling:` 兩行。`# Big Story` 用 bullet，每個 beat 一行：

```
* [B5_field_work|turn] <一段有畫面、有節奏、有聲音、有情緒落點的散文> {feeling: peak, proud}
```

**寫每個 beat 的硬要求——把抽象換成感官：**

| 不准這樣寫（鬼藍圖） | 要這樣寫（有靈魂） |
|---|---|
| 基本技能：高密度展示剪鐵線、繩結法 | 一張張手的特寫接著切——剪鐵線、打繩結、洗礙子，越來越快，像在炫技，配樂踩著拍點往前推 |
| 主任期勉：主任勉勵叮嚀 | 音樂收小，只剩主任的聲音。鏡頭給他一個穩穩的特寫，中間穿插學員咬牙拼搏的高光，他說的話讓畫面有重量 |
| 大合照，宣示結訓完成 | 全班坐滿球場，鏡頭拉開、定住。這一張要久一點，讓人看清楚每一張臉。它是證據，是「我們真的做到了」 |

每個 beat 的 prose 至少要讓人讀出：**它看起來像什麼（意象）、它多快或多慢（節奏）、
該聽到什麼（原音）、它的情緒往哪去（落點）**。這四樣藏在文字裡，不用標籤，但一定要在。

> 範例對照（開場）：
> 鬼藍圖「回憶啟航：最後的實習報告與訓練生活回顧」
> 有靈魂：「片子從一支筆開始。一個學員低頭在實習報告上寫字，筆尖特寫、窗外的光、
> 還沒收完的行李——一切都還很慢、很安靜，像有人在翻一本快闔上的相簿。最後落在四個
> 手寫字『人生的 0.66』。」

## 4. 靈魂完整性守門（出 blueprint 前必過，這是「硬」的地方）

逐 beat 檢查，**任何一項缺失就退回重寫該 beat，不准出**：

```
□ 情緒型態可讀     prose 能對應到一個 content_pattern（emotional/action/enumeration…）
□ 節奏可讀         有密度語感（快/疊上來）或留白語感（停住/慢慢看）至少一種
□ 原音意圖         凡有人講話/現場動作的段，講清楚原音要不要留
□ 至少一個感官意象  有具體畫面/小物特寫，不是只有抽象類別名詞
□ 勾回 thesis      這段的情緒/動作能連回中心句（意境一致）
```

整片層級：

```
□ thesis 存在且是一句「在說什麼」，不是題目（「結訓影片」不算 thesis）
□ 情緒弧完整：有安靜的起、有堆疊的承、有爆/轉、有收
□ bookend 具體：開場第一鏡、收尾最後一畫面都寫出來了
□ 必放 🔒 都標記，且註明不可被 stock/生成頂替
□ anti_goals 至少 1–2 條
```

守門失敗時，**指名是哪個 beat 缺哪一項，回去問那一題**（用 §2 的對應問題），
不要自己猜著補——猜出來的就是沒靈魂。

## 5. 寫 blueprint.json（thin index，the handle）

從 blueprint.md 抽出，給下游引用。**只放 meaning，不放 mechanics**（無 provider/檔案/時間）：

```json
{
  "artifact_role": "narrative_blueprint",
  "version": 1,
  "thesis": "人生的 0.66% 是一段用汗水攀爬、用技術立足的紮實路程，我們在此寫下回憶與感謝。",
  "intended_feeling": "warm, quietly proud",
  "mode_hint": "warm_documentary",
  "beats": [
    { "id": "B5_field_work", "role": "turn",
      "summary": "現場實務：爬桿拖纜換礙子，引擎全開後在攻頂那刻留白",
      "intended_feeling": "peak, proud", "prose_ref": "Big Story B5" }
  ],
  "anti_goals": ["避免冗長無節奏的呆板操作畫面", "特效不要誇張，維持現場的莊嚴專業"]
}
```

```
beats[].id 是穩定錨點，下游 segment.core.blueprint_ref 會引用它（forward/backward gate）。
summary 是給機器的索引，靈魂在 blueprint.md 的 prose，不要把 prose 壓進 summary。
```

> ⚠️ thesis 抽取注意：`blueprint_compile.py` 目前用句號斷句，會把「0.66%」的小數點當句號
> 截斷 thesis。產 blueprint.json 時請保留完整 thesis 句（或先確認該 bug 已修），
> 別讓中心句在第一步就斷頭。

## 6. 鏈位置與交接

```
模糊需求
  └ blueprint-interview（本技能）: 引導 → blueprint.md（soul）+ blueprint.json（index）
        ↓  下游照 imagery-to-edit-lexicon 翻譯
     director / spec-contract: blueprint → segment_contract.json
        （content_pattern / sequence_grammar / pacing / audio / text_layer）
        ↓
     material_treatment.resolve_treatment + shot_slots.expand_shot_slots（引擎已存在）
        ↓
     editor_review / visual_fatigue / treatment_audit / editorial_qa
        （驗 beat_coverage / 頻率 / 留白 / 原音 是否忠實執行）
```

**Skill handoff 慣例：**
- `Upstream`：使用者的模糊委託（+ 選擇性的 material-map）。
- `Owns`：可改 thesis/情緒弧/各 beat 的 prose 與 feeling；可決定意境與反目標。
- `Downstream（必產）`：`blueprint.md` + `blueprint.json`，且過 §4 守門。
- `VERIFY hooks`：Node 11 `beat_coverage`（dropped_beat 阻擋）、Node 12 `editorial_qa`
  的 thesis_alignment / arc_order / anti_goal_respect。

## 附錄：一段完整引導示範（從模糊到具體）

示範「不接受模糊、追到具體」的真實深度。委託＝「幫我剪 66 期結訓影片，熱血感動一點」。

```
Q（定靈魂）：這幾個月對這群人來說，如果濃縮成一句話，是什麼？
A：就……很操，但學到很多，大家感情很好。
Q（追問 thesis）：那「很操但值得」——你想讓看的人最後心裡留下哪一個？驕傲？不捨？感謝？
A：應該是……驕傲又感謝。這段日子在人生裡很短，但很重要。
→ thesis 收斂：「人生的 0.66%——一段很短卻刻骨的路，用汗水換來，值得驕傲與感謝。」（委託人點頭）

Q（情緒弧）：哪一段最操、最用力？哪一刻你會鼻酸？
A：現場實務最操，爬桿拉線那種。鼻酸是最後大合照跟離營吧。
→ arc：安靜開場 → 訓練堆疊 → 現場實務爆點(turn) → 課餘放鬆 → 結訓收束(resolve)

Q（質地·現場實務）：那段你看到的第一個畫面是什麼？
A：就大家在爬電桿啊。
Q（追問 detail）：爬桿的哪個瞬間？有沒有一個近的、看得到費力的畫面？
A：喔有，汗一直滴、手套抓著桿，還有最後線接上去那下。
Q（追問 節奏）：這段是一個接一個快切讓人喘不過氣，還是慢慢看？
A：快！要那種很多畫面狂切的感覺。
Q（追問 原音+留白）：該聽到什麼？還有線接上那一下，要停一下嗎？
A：要聽到現場喊口令、金屬聲。對，接上那下可以停一下，很有成就感。
→ 這個 beat 的判準全填滿了：action / fast→hold / 口令+金屬聲keep / 汗珠+手套+線接上 / 勾回驕傲 ✓

Q（必放🔒）：一定要在裡面的人或時刻？
A：所長早會宣誓、主任講話、各班大合照、謝師。
Q（誠實確認）：這幾個能不能用別的畫面代替？
A：不行，一定要本人跟真的合照。
→ 標記 testimony/proof🔒，不可頂替。
```

寫出來的 B5 beat（注意每個判準都化進文字，沒有一個是標籤）：

